1) Elaborate on a few (3-5) different security issues or concerns, from your own experience, related to web application security and relate them to the OWASP top ten? Answer: from my own experience I can mention SQL injection, XSS or cross-site scripting and sensitive data exposure. The XSS is basically when you accept data from the user, and you are displaying it in your web page, and you are not escaping it. For instance let’s say you had taken some data from the user and then you return it to the user in the text area and if the code that actually they typed in has some HTML in it and especially if you’ve used "script" tag and they can actually put code in the script area that might fetch all the cookies using document. Cookies and make a request to another URL (sending your cookies there). And if I view this page which has other content then it could cause the browser to load my cookies and send it to some other site then the other user could then look at that request and put his cookies in his browser and then start browsing the site as me. But if you escape the HTML you don’t have to worry about it. Injection: for example, the input from a user is //select * from person where name = Hamid or 2 = 2 which causes that every row from the person table in the DB would be returned back and that is not good, and this is called SQL injection. What would happen is that we are taking the input from the user and use that input directly in a query without validating it and the best to resolve that is using parameterized queries so we use? mark and make sure that the value in its entirety is passed into the name. and here is how the query gets executed: //select * from person where name = ‘Hamid or 2 = 2’ and then this will check this as a name. 2) Summarize by providing your own guidelines to a new web programmer on the topic “The essential guidelines to follow to create secure web applications”. Answer: 1- Implementing a secure SDLC framework for software development.2- gain the knowledge and appreciation for OWASP and build skills for threat modeling. 3- capture security requirements of the applications right at the start of the software development lifecycle. 4- implement secure application design and architecture. 5- ensure secure coding practices r input validation, authentication and authorization. 3) What is your TIL for this course section?
